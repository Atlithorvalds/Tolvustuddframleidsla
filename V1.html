VERKEFNI 1 Í TÖLVUSTUDDRI FRAMLEIÐSLU - VEFSÍÐUGERÐ

Undirbúningsvinna og kynning á Brackets

Fyrsta verkefni annarinnar var að búa til vel skipulagða vefsíðu sem heldur utan um öll verkefnin sem verða unnin í áfanganum VÉL608G. Það fyrsta sem ég gerði var að lesa vinnulýsinguna á Canvas. Eftir það fór ég að hugsa um það hvernig ég ætlaði mér að hanna og setja upp vefsíðuna. Ég byrjaði því á að horfa á fyrsta kennslumyndbandið. Þegar ég var búinn að horfa á öll fjögur kennslumyndböndin, á fann ég aftur vinnulýsinguna og byrjaði að ná í öll forritin sem ég þurfti Brackets og GitHub. Ég hafði nú þegar kynnst aðeins notkun á HTML í öðru en Brackets en ákvað að nota það sem kennarinn mældi með. Kennarinn nefndi meðal annars að kostir við Brackets væru svokallaður "Live" eiginleiki og eins "Git integration". Ég setti forritin upp og byrjaði að vinna með þau og reyna að læra alla helstu kosti og galla þeirra. Ég fór að skoða vefsíðusniðmát og valdi Spectral frá Html5Up. Næst var að kynnast Brackets betur og að byrja að setja vefsíðuna upp, með það markmið að hún henti sem vinnubók áfangans og í framhaldinu portfolio.

Uppsetning á GitHub og að koma síðunni inn á netið

Þegar að vefsíðan var að verða tilbúin var næsta skref að hlaða henni upp í Github. Þar sem ég var ekki með aðgang að Github byrjaði ég á því að búa mér til aðgang. Þar á eftir horfði ég á GITHUB myndband1 á Youtube og fór eftir þeim leiðbeiningum sem komu fram í myndbandinu en ég mun fara yfir það sem ég gerði í nokkrum skrefum.

  1. Ég byrjaði á að setja upp GitHub og fara inn á Github aðganginn minn. Þar setti ég upp New repository á Github síðunni minni undir nafninu Tolvustuddframleidsla. Þann hnapp má finna undir plúsinum í hægra horninu á Github síðunni.

  2. Næst þarf að fylla inn í gluggann Repository name og passa að það sé hakað í Public hnappinn en það er svo að aðrir geti séð vefsíðuna. Þegar þetta er komið er ýtt á hnappinn Create repository.

  3. Næst þarf að hægri-smella á möppuna sem vefsíðan okkar er geymd í og það þarf að velja New terminal at folder og þá opnast terminal gluggi. Við förum þá aftur inn á Github og við þurfum að afrita textann á síðunni sem opnaðist inn á terminal gluggann.

    Ég reyndi að nota eftirfarandi skipanir í Git Bash:

    echo "Tolvustuddframleidsla" >> README.md
git init
git add README.md 
git commit -m "first commit"
git branch -N main
git remote add origin https://github.com/Atlithorvalds/Tolvustuddframleidsla.git
git push -u origin main

    Þetta gekk ekki vandræðalaust fyrir sig og fékk ég villukóðann:

    error: src refspec main does not match any
error: failed to push some refs to 'https://github.com/Atlithorvalds/Tolvustuddframleidsla.git'

    Eftir goggle leit þá fann ég á Stack Overflow lausn en sú lausn voru eftirfarandi tvær skipanir:

    git commit -m 'fixes' --allow-empty
git push
    Eftir það þurfti ég að setja allar skipanirnar aftur inn: 
    echo "Tolvustuddframleidsla" >> README.md
git init
git add README.md 
git commit -m "first commit"
git branch -N main
git remote add origin https://github.com/Atlithorvalds/Tolvustuddframleidsla.git
git push -u origin main

    Þegar búið er að afrita kóðann inn á terminal gluggann og ýta svo á enter. Ef allt gengur upp, þá þarf að skrá notendanafn og lykilorð (Token) inn. Þar lenti ég í vandræðum þar því ég var ekki búinn að kynna mér og búa mér til Token. Ég fann út úr því með því að fara eftir leiðbeiningum sem ég fann á þessari síðu.   4. Nú ættu allar skrár að vera komnar inn á Github síðuna og ætti vefsíðan þá að líta svona út.

    Þá förum við í settings í hægra horninu og þar förum við í GitHub pages. Þar breytum við None í Main og vistum breytingarnar og þá ætti vefsíðan að líta svona út:
    Nú er hægt að nota þennan link sem birtist til þess að deila síðunni með öðrum.

    5. Enn var hægt að gera breytingar á þessari síðu og var það fremur einfalt. Allt sem þurfti að gera var að opna aftur New terminal at folder og líma skipunina: 'git add . ; git commit -m"BreytingNr"; git push'.

    Að AÐLÖGA VEFSÍÐUNA AÐ MÍNUM ÞÖRFUM og Þörfum námskeiðsins

    Ég vildi hafa vefsíðuna á ákveðinn hátt og því þurfti ég að skipuleggja mig svolítið. Ég þurfti að hugsa um sniðmát, textastærð, litaval, ofl. Þegar ég var búinn að ákveða var þetta einfaldlega gert með því að prufa sig áfram í kóðanum. Sumt var lítið mál eins og að breyta litnum á bakrunni. Það var gert með því að velja í kóða: "wrapper style4" í stað "wrapper style5" sem dæmi. Eins notaði ég mér það að ég var með tilbúna síðu í höndunum og gat því einfaldlega breytt kóðanum og ef það klikkaði eða mér leist ekki á breytingarnar þá einfaldlega breytti ég því til baka. Svo það var voða lítið mál að læra á hvað gerði hvað. Til dæmis hvernig maður átti að gera bil og/eða línu sem skiptir texta. Það er allt voða einfalt þú notar það sem þú sérð í kóðanum og afritar það á þann stað sem hentar. Hlutir eins og að stækka og minnka mynd gerði ég með því að stækka eða minnka hana í Preview, sem er forrit í tölvunni ferð bara í Adjust size og lagar hana til þangað til hún kemur vel út. Þetta lýsir nokkuð vel hvernig ég lagaði til allt á vefsíðunni. Ég prufaði mig bara áfram og lærði hægt og rólega hvernig kóðin virkaði. Þar aðleiðandi notaði ég ekki mikið af síðum til að finna upplýsingar. Það er hins vegar eitthvað sem ég mun eflaust gera þegar ég fer í að bæta vefsíðuna enn frekar.

    FERILSKRÁ

    Ég ákvað að hafa ferilskrána sem síðu á vefsíðunni þar sem það var bæði þægilegt og eins einfalt að laga ef einhver villa kom upp í stað þess að hafa hana sem pdf og þurfa þá að setja inn nýtt og nýtt skjal. Gerð ferilskránar var ekki flókin ég einfaldlega tók það helsta úr minni ferilskrá og bætti einu og einu við. Þegar ég var orðinn nokkuð ánægður með hana og útlitið á henni, þá lagaði ég hana aðeins til og sagði það gott.

    Það sem ég vil fá út úr námskeiðinu

    Það sem ég býst við og vil fá út úr þessu námskeiði, er í raun það sem hann fjallar um. Að eiga mína eigin vefsíðu, læra á alla þessa "nýju" tækni og verða vonandi sæmilega seigur að vinna með þessa tækni og tæki. Því þessi tæki eru framtíðin og að kunna á þau og geta skapað eitthvað í þeim er gríðarlegur kostur að hafa. Það mun einnig hjálpa manni á atvinnumarkaðnum. Fyrir utan allt þetta er þetta ekkert annað en skemmtilegt og að hafa tækifæri á að prufa og að læra á þessi tæki eru forréttindi. Ég vona því að þetta gangi vel og að maður komi betri nemandi og starfsmaður út úr þessu námskeiði.
